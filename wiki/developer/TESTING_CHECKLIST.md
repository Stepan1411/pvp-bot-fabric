# Testing Checklist - PVP Bot API v1.1.0

## Перед тестированием

- [ ] Скомпилирован проект: `./gradlew build`
- [ ] Нет ошибок компиляции
- [ ] Версия API обновлена до 1.1.0

## Тест 1: Combat Strategy Integration

### Подготовка
- [ ] Создана тестовая стратегия с логированием
- [ ] Стратегия зарегистрирована в `onInitialize()`
- [ ] Приоритет установлен на 100+

### Выполнение
- [ ] Запущена игра
- [ ] Спавнен бот: `/pvpbot spawn TestBot`
- [ ] Боту дана цель (спавнен моб рядом или игрок)

### Ожидаемые результаты
- [ ] В консоли появляются сообщения `canUse() called`
- [ ] В консоли появляются сообщения `execute() called`
- [ ] Сообщения появляются каждый тик когда бот атакует
- [ ] Если `execute()` возвращает `true`, стандартная атака не выполняется

### Проверка приоритета
- [ ] Зарегистрированы 2 стратегии: одна с приоритетом 200, другая с 100
- [ ] Стратегия с приоритетом 200 вызывается первой
- [ ] Если первая стратегия возвращает `true`, вторая не вызывается

## Тест 2: Attack Handler Cancellation

### Подготовка
- [ ] Зарегистрирован обработчик атаки
- [ ] Обработчик возвращает `true` для определенных целей

### Выполнение
- [ ] Спавнен бот
- [ ] Спавнена цель, для которой атака должна быть отменена
- [ ] Бот получил эту цель

### Ожидаемые результаты
- [ ] В консоли появляется сообщение от обработчика
- [ ] Бот машет рукой
- [ ] Цель НЕ получает урон
- [ ] Бот продолжает следовать за целью

### Проверка разрешения атаки
- [ ] Спавнена другая цель, для которой атака разрешена
- [ ] Обработчик возвращает `false`
- [ ] Бот наносит урон цели

- [ ] Стратегия с использованием `server` параметра работает
- [ ] Стратегия с использованием `bot.getEntityWorld()` работает
- [ ] Нет runtime ошибок

## Тест 4: Multiple Strategies

### Подготовка
- [ ] Зарегистрированы 3 стратегии с разными приоритетами:
  - Стратегия A: приоритет 200, всегда возвращает `false`
  - Стратегия B: приоритет 150, возвращает `true` при HP < 50%
  - Стратегия C: приоритет 100, всегда возвращает `false`

### Выполнение
- [ ] Спавнен бот с полным HP
- [ ] Бот атакует цель

### Ожидаемые результаты (полное HP)
- [ ] Вызывается стратегия A (`canUse()` и `execute()`)
- [ ] Вызывается стратегия B (`canUse()` возвращает `false`)
- [ ] Вызывается стратегия C (`canUse()` и `execute()`)
- [ ] Выполняется стандартная атака

### Ожидаемые результаты (низкое HP)
- [ ] Вызывается стратегия A
- [ ] Вызывается стратегия B (`canUse()` возвращает `true`, `execute()` возвращает `true`)
- [ ] Стратегия C НЕ вызывается
- [ ] Стандартная атака НЕ выполняется

## Автоматизированное тестирование

Для автоматизации тестов можно использовать:

```bash
# Запустить все тесты
./gradlew test

# Запустить конкретный тест
./gradlew test --tests "CombatStrategyTest"

# Запустить с подробным выводом
./gradlew test --info
```

## Continuous Integration

Добавьте в `.github/workflows/test.yml`:

```yaml
name: Test PVP Bot API

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Set up JDK 17
        uses: actions/setup-java@v2
        with:
          java-version: '17'
      - name: Build with Gradle
        run: ./gradlew build
      - name: Run tests
        run: ./gradlew test
```
